# Run DC Drift Verification

Goal: Verify the implementation of a UI screen against its approved Figma design.

Steps:

1. Retrieve the approved Figma frame.
2. Capture the implementation screenshot.
3. Compare both using visual reasoning.
4. Apply the DC Drift rules from rules.md.
5. Generate findings and Drift Score.

Return:

1. Drift Score
2. Issue list
3. Severity counts
4. Release recommendation
5. Three reports:

* PM summary
* Engineer checklist
* Designer report
